本项目的介绍：情况一：不打开其他游戏（你现在就可以这样做）
你的应用完全可以独立运行，不需要依赖任何其他游戏。应用启动后：
它会显示当前的设备温度（可能是室温或待机温度，通常在 30°C 左右）
“性能生存挑战”小游戏会正常运行（得分可能增长较慢）
所有按钮和功能都可以正常使用
适合场景：开发调试、演示应用功能、测试UI交互。
情况二：同时打开一个游戏（推荐测试方式）
如果你想测试真实的游戏场景感知效果，建议：
先打开你的应用，点击“初始化场景感知”
然后切换到后台（不要关闭应用）
打开一个大型游戏（如《王者荣耀》《原神》等）
玩几分钟后，再切回你的应用
会发生什么：
游戏运行时手机会发热 → 温度等级升高 → 你的应用会检测到温度变化
“性能生存挑战”得分会因为温度升高而下降（模拟真实游戏体验）
应用会给出“建议降低画质”的提示
为什么这样设计？
你的应用是一个“游戏辅助工具”，它的核心价值在于：
实时监控：像游戏伴侣一样，在后台默默监控设备状态
智能提醒：当设备过热时，提醒玩家调整设置
数据可视化：把看不见的温度、性能数据变成看得见的图表和建议
实际操作建议：
第一次运行：直接运行你的应用，熟悉功能 ✅
第二次测试：
打开你的应用
点击“初始化场景感知”
点击“订阅设备状态”
按 Home 键回到桌面
打开《王者荣耀》玩一局
切回你的应用，看温度变化和日志
小技巧：
如果你不想真的玩游戏，也可以：
边充电边用手机：充电会让设备发热，模拟游戏时的温度上升
同时运行多个应用：多开几个应用也会让CPU升温
使用性能测试工具：如安兔兔跑分，会让设备满载运行
注意事项：
权限问题：确保你的应用有“后台运行”权限，否则切到游戏后可能被系统杀掉
省电策略：在手机设置里把你的应用设为“不限制后台运行”
温度检测：有些手机的温度传感器比较保守，可能需要玩一会儿游戏才会明显升温
总结：你的应用是个独立的工具，不依赖其他游戏也能运行。但要想体验它真正的“游戏场景感知”能力，配合一个真实游戏一起使用效果最好！
